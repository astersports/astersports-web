SCOPE / SAFETY (resolved guardrails)

  No prod, no money path, no flip, no UI.
- DATA EGRESS: G2 obligations attach at EVAL time (the image leaves to Replicate),
  not just flip time. RESOLUTION: the pilot uses OWNED / FIXTURE / SYNTHETIC garments
  ONLY — NO real customer images — so no customer data egresses pre-G2. (G2 extends
  to the SAM3 endpoint as a flip-time gate, separately, before any real garment.)
- ENDPOINT: `mattsays/sam3-image` (Replicate) — confirmed STILL-IMAGE SAM3 path
  (~$0.0013/run, ~2s, L40S), text + exemplar + box/point/mask prompts. Community-
  published, but runs under Replicate's PLATFORM data-retention: API predictions'
  inputs/outputs auto-delete after 1h (web-UI predictions kept indefinitely — we use
  the API). This is PARITY with our existing SAM2-on-Replicate posture, not new
  exposure. [GROUNDED/web 2026-06-28: replicate.com/mattsays/sam3-image;
  replicate.com/docs/topics/predictions/data-retention]
- Pin model VERSION (pinned Replicate version hash) + seed; record both in every row.

FIXED PROMPT STRATEGY (declared up front — no per-image cherry-picking)
- ONE text-concept prompt per garment CLASS, set in the manifest BEFORE the run and
  applied uniformly (e.g. floral set → "flower"; geometric set → "printed motif").
  The per-class prompt is frozen in the manifest; it is never re-chosen after seeing
  any output. Exemplar prompting is NOT used in round 1 (keeps it reproducible +
  comparable). If a class needs a different fixed prompt, it is declared in the
  manifest before the run, with rationale, and applied to ALL images in that class.

ACCEPTANCE BAR — SAM3 ships as the candidate ONLY IF ALL pass (locked):
[ ] A1 COUNT ACCURACY — on the fixed runners vs ground-truth instance counts, SAM3
       count MAE ≤ 0.75 × SAM2 count MAE (≥25% relative improvement) AND the gap
       exceeds the run-to-run noise band measured in A3. (A real margin, not noise.)
[ ] A2 NO FAILURE-MODE REGRESSION — SAM3 does not regress the giant-instance / over-
       segmentation mode: (giant-instance incidence = an instance > MAX_INSTANCE_
       FRACTION 0.20 of fabric) and (over-segmentation rate = predicted/truth count
       ratio) are each ≤ SAM2's on the same set.
[ ] A3 COUNT STABILITY (billing-contract test) — under the pin (version+seed), K≥5
       repeat inferences on each garment return the IDENTICAL instance count every
       time (100% identical). Any count drift under pin = FAIL for the money path.
[ ] A4 MASK STABILITY (cache-moat test) — under the same pin, repeat masks are
       pixel-stable: mean per-instance IoU across the K runs ≥ 0.99, and report the
       byte-identical rate. Determines the freeze-on-bill requirement (see RULING).
[ ] A5 SCALE FALSE-REJECT — on the scale repeat set, SAM3-fed detection improves the
       false-reject rate vs the current SAM2-fed detector (the CLAUDE.md §4 G3 gap).

DETERMINISM RULING (the moat — architect-set)
- SAM3 may be a MONEY-PATH provider only if it slots into the SAME reproducible-by-
  cache pinning SAM2 uses: cache key MUST incorporate provider + model-version + seed,
  and once a garment is billed its mask is FROZEN IN STORAGE, never recomputed.
- If A3 passes but A4 fails (count-stable, not mask-stable): still viable ONLY GIVEN
  freeze-on-bill, AND the eval/design must prove the recompute-on-cache-miss path
  cannot silently change an ALREADY-BILLED customer's output (i.e. a cache miss after
  billing re-reads the frozen mask, never re-infers). State this explicitly in the
  return doc.
- If A3 FAILS (not count-stable under pin): SAM3 CANNOT touch the money path at any
  accuracy — it could only ever feed a QUARANTINED non-count path. Report and stop.

BASELINE
- Re-run SAM2 FRESH on the same runners in the same harness pass — NO stale recorded
  numbers. Apples-to-apples, both providers, same garments, same metrics, same run.

HARNESS BUILD (what CC implements — offline, mirrors sam2 eval)

3. Metrics: count MAE, giant-instance incidence, over-seg ratio, K-repeat count

4. Output: a comparison TABLE (SAM2 vs SAM3 per metric) + per-garment rows +
