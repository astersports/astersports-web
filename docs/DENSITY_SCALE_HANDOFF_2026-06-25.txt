2. THE STRATEGY (decided — do not re-derive; full text in DENSITY_SCALE_STRATEGY.txt)
================================================================================
2.1 The client's catalog splits into FOUR surface buckets (product names are diagnostic):
  A. SCATTERED all-over prints ("tossed/trailing floral", dots) — artfully RANDOM, NOT a
     geometric repeat. High volume. → DENSITY'S SWEET SPOT.
  B. PLACED embellishment / sequins / paillettes (brand signature) — hard case.
  C. SOLIDS — density/scale DO NOT APPLY (UI must suppress the controls).
  D. LACE.

2.2 The key insight that reorganizes everything:
  Their prints are SCATTERED, not periodic. That splits the two ops in OPPOSITE directions:
  - DENSITY loves scattered: counting motifs + removing an exact fraction is
    periodicity-agnostic. Their florals are ideal for density.
  - SCALE chokes on scattered: the old repeat guard (FFT + autocorrelation) looks for a
    PERIODIC tile; a tossed floral has no strict period → it gets FALSE-REJECTED (~1/3 of
    genuine repeats — the CLAUDE.md §4 G3 complaint). FIX (Phase 2): scale must treat
    "all-over scattered coverage" as scalable and scale by RESIZING MOTIFS, not re-tiling.

2.3 The architecture (combine, don't pick):
  - FORMULA/DETERMINISTIC = ACCURACY. Only it can count + place EXACTLY ("remove exactly
    50%"). Billable, reproducible. The spine.
  - GENERATIVE AI = REALISM. Only it makes a hard garment look photographically right.
  - THE TRICK: run the generative pass MASKED so it polishes the LOOK without changing the
    COUNT/GEOMETRY. Route the finish to the garment: solid ground → cheap formula fill;
    busy/complex ground or on-model → constrained generative finish.

2.4 FLUX.1 Fill spike (PR #104) — EVALUATED, NOT ADOPTED (closed/parked). FLUX ≈ the
  simple fill on solid grounds (no visible win), ~100x cost, new sub-processor + paid
  licence. Reopen only if a client garment has a genuinely busy/photographic ground.

5. THE SAM2 GO/NO-GO MEASUREMENT (docs/SAM2_COUNT_EVAL.md) — THE DECISIVE RESULT
================================================================================
Question: does the REAL production SAM2 count path hit ±10% on the hard conditions where
classical hit 40%?  Measured via a CI workflow (creds via the REPLICATE_API_TOKEN secret;
sentinel-triggered): autoSegment(dataUrl, params) → finishSam2Segmentation(seg, meta) →
DERIVED instances (production derivation: specks <0.02% dropped, giants >20% crop dropped
as ground, cap STUDIO_MAX_INSTANCES=200) → count. NOT raw mask count. Best of pps16/32/64,
no per-image cherry-picking, no fitting to N. The vision-LLM fabric LOCATE is bypassed (full
frame for synthetic, manual bbox for reals) so it measures SAM2's COUNTING, not the locator.

RESULT — SYNTHETIC (rigorous, exact N): SAM2 = 10/10 within ±10%. Per condition:
  heavy-overlap   classical 23% → SAM2 8%
  multicolor-bunch classical 23% → SAM2 3%
  two-tone ground  classical 63% → SAM2 2%
  gradient ground  classical 12% → SAM2 3%
  low-contrast     classical 100% (blind) → SAM2 2%;  contrast-mid 100% → 0%
  (separated/high-contrast controls all 2-3%)

VERDICT: BILLING-GRADE DENSITY IS ACHIEVABLE — with SAM2 as the count source. SAM2 does
what classical fundamentally cannot: split touching motifs, and see low-contrast / varied-
ground prints. This is the go.

CAVEAT (do not skip): the REAL-floral rows are NOISY and NOT trustworthy yet —
  poppette SAM2 45 vs classical 93 (UNDER), pindot 1 (found ~nothing), tossed 143,
  tessa 211, stassie 130 — because (a) no firm hand ground-truth, and (b) the eval fed a
  MANUAL bbox crop, not the production vision-LLM locate. So the real-image path needs the
  full production locate + crop + per-route tuning before billing. The trustworthy signal
  is the synthetic known-N bar (10/10).

APPENDIX B — densityThin ALGORITHM (the billed op; densityPreview calls it per percent)
================================================================================
Inputs: image{url}, fabric{bbox, raster(REQUIRED), boundaryRaster?}, instances[] (each
{bbox, raster}), percent (0..90), useLama?. Output: {data(RGBA), width, height, removed}.
Steps (server/_core/studio/ops/densityThin.ts):
  1. decodeUpright(image.url) → buffer/w/h. Require fabric.raster and dims match (else throw).
  2. n=instances.length; removeN=clamp(round(n*percent/100),0,n). If n==0||percent<=0||
     removeN==0 → return removed:0 (passthrough).
  3. DIM-DRIFT GUARD (F3): if any instance.raster missing or != image dims → return
     removed:0 (refund; never bbox-paint a paid op).
  4. selected = stratifiedSelect(instances, removeN, fabric.bbox, w, h)  [even spatial spread].
  5. baseClothLab = dominant Lab cluster of bare ground (in-fabric, not under any instance,
     k-means k=3 seed=1). If none (fully covered/empty) → return removed:0 (F2 refund; never
     smear black).
  6. region = selected instances, DILATED 2px, clipped to raster AND to NOT any survivor
     instance (survivor-clip). If regionCount==0 → return removed:0 (F1 refund).
  7. infill: useLama&&available → lamaInfill(region) (texture), else infillBaseCloth(region,
     baseClothLab, featherPx:1, flatten:true) (flat LAB — replaces L too so opaque motifs
     don't leave a luminance ghost).
  8. Restore every SURVIVOR pixel to the original RGB (feather can graze a shared edge) →
     survivors stay byte-identical.
  9. return removed:selected.length.
Invariants: pure + deterministic; survivors byte-identical; removed reflects EFFECT (never
bills a no-op). densityRedistribute.ts (v2) additionally RELOCATES survivors (blue-noise) —
flag-gated, it was the "blob" source; v1 thin-in-place is the safe default.

APPENDIX C — repeatDetector DECISION (server/_core/studio/ops/repeatDetector.ts)
================================================================================
detectRepeat(imageRgba, w, h, rasterData) →
  bb = maskBBox(raster); if <16px → "insufficient_data".
  xRes=analyzeAxis(xSignal), yRes=analyzeAxis(ySignal)   [FFT power spectrum on masked
    luminance; peakRatio>=0.30 & periodicityEnergy>=0.50 & harmonicPeaks>=2 & tileRepeats
    >=2.5 & autocorr>=0.25 → axis passes].
  passing = axes that pass.
  IF passing==2 → isAllover=true, classification "allover", acceptPath "periodic".
  ELSE IF passing==1 → isBorder=true, "border", REJECT (FFT is the border guard) + attach
    coverage diag.
  ELSE (passing==0) → cov=analyzeAllover(...); IF cov.isAllover → "allover", acceptPath
    "coverage"; ELSE "placement", REJECT.
analyzeAllover(...):
  ground=median Lab over sampled masked px. For each masked px in bb: fg if Lab-dist to
    ground > ALLOVER_FG_TAU(12). Also accumulate mean |dL| (edgeEnergy).
  REJECT (return !isAllover) if foregroundFrac < 0.012 or > 0.96 (empty/near-solid), or
    edgeEnergy < 0.4 (smooth gradient/solid — not a print).
  CC the fg (4-connectivity, so corner-touching motifs stay distinct) within bb; keep comps
    with area >= max(4, 0.00004*bboxArea). numMotifs = kept.length.
  Grid 5x5: cell occupied if cellFgFrac > 0.008. occupancy = occupied/cells; activeRows,
    activeCols = distinct grid rows/cols with an occupied cell. medianSpan = median of
    max(w,h)/bbox over kept comps (reported, not gating).
  isAllover = numMotifs>=2 AND occupancy>=0.60 AND activeRows>=4 AND activeCols>=4.
checkRepeatAdvanced(...) = thin wrapper returning {isRepeat:isAllover, confidence,
  classification, axes}. aiEngine.generateScaledImage rejects (NON_REPEAT_SCALE_ERROR)
  unless isRepeat.

APPENDIX D — SAM2 INSTANCE DERIVATION (what "count" means; sam2Provider.ts + sam2Mask.ts)
================================================================================
Production path (async worker): vision-LLM locate (locateFabricRegionForDensity) → crop to
fabric bbox (Req-1 crop-to-fabric minimization) → autoSegment(dataURL) → {combined_mask,
individual_masks}. finishSam2Segmentation(seg, meta) builds:
  fabric.raster = FULL CROP filled 255 (base-cloth SAMPLING mask; combined_mask is too
    sparse on dense prints) + boundaryRaster from combined_mask (garment silhouette).
  instances = instancesFromMasks(individual_masks)  [each mask decodeMaskToRaster; drop
    specks < MIN_AREA_FRACTION 0.0002 of crop + abs floor] → instancesFromSegment filter:
    DROP any instance > MAX_INSTANCE_FRACTION (0.20) of crop (that's ground, not a motif) →
    CAP at STUDIO_MAX_INSTANCES (200) largest → remap each to full-image coords.
The COUNT the density op consumes = instances.length AFTER that derivation (NOT raw
individual_masks length). SAM2 model: meta/sam-2 @ version fe97b453…dc83 (community model
→ must run by version, not slug). Input = base64 data-URL (privacy-minimal). Tunables:
points_per_side (default 16), pred_iou_thresh 0.82, stability_score_thresh 0.88, use_m2m.
Missing REPLICATE_API_TOKEN → MaskProviderUnavailableError → fail-safe to classical (which
has NO instances → density fails+refunds; density never prompt-falls).

APPENDIX E — METRIC FORMULAS (eval/phase1Metrics.ts, eval/phase1Audit.ts)
================================================================================
M1 density fidelity (per case at percent p): actual = removed/totalTruthInstances;
  PASS if |actual - p/100| <= 0.10*(p/100)  (i.e. ±10% RELATIVE; "ask 50% → 45–55%").
M2 segmenter detection: relErr = |detectedCount - trueN| / trueN; PASS if relErr <= 0.10.
  Gating bar = clean synthetic only; dense/overlapping reported as ceiling ("needs SAM2").
M3 scale recall/safety: per labeled case, accept = detectRepeat(...).isAllover.
  recall = (#accept among want-accept) / (#want-accept); REQUIRE >=0.90.
  false-accepts = #(want-reject AND accept); REQUIRE == 0 (the anti-"split-in-two" safety).
SAM2 eval (eval/sam2CountEval.ts): same M2 formula, but detectedCount = SAM2 DERIVED
  instances (Appendix D), best of pps16/32/64 per condition.

APPENDIX I — GLOSSARY
================================================================================
  motif/instance = one printed element (a flower, a dot). density counts these.
  all-over / scattered = print covers the fabric with random (aperiodic) placement.
  placement = a single placed graphic/logo (NOT scalable as a repeat → reject).
  border = periodic on one axis only (reject).
  derived instances = SAM2 masks AFTER speck/giant filter + cap (the billable count).
  classical = the offline contrast+CC segmenter (no creds; the stand-in with a real ceiling).
  G3 = the CLAUDE.md launch gate: real-garment per-route eval before a *_LIVE flip.
  no-op / degrade = nothing erased (removed==0) → MUST NOT bill.
