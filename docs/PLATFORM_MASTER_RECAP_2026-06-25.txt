The audit cycle found two different KINDS of problem, and the second one is the one
that actually threatens the business:

      routine, the worst clustered in St Patrick. Largely FIXED this session for the

      Frank: THREE of the five repos (astersports-web, st-patricks-armonk, legacy-

    UNAFFECTED. The live youth-sports app and the Legacy Hoopers tenant data are fine.
    Everything merged there this session IS live. [GROUNDED]
  · astersports-web (astersports.io / Print Studio) — the off-Manus migration #103
    (→ Railway + Supabase + Anthropic + Google OAuth) HAS NOW MERGED to main this
    session [GROUNDED: origin/main top commits show #103, #114, #115]. So its migration
    is no longer "not started" — the code landed. OPEN: is it actually deployed + serving
    on Railway, with all env/secrets/DB set? [VERIFY]
  · st-patricks-armonk — the WORST-positioned: a LIVE CLIENT deliverable holding
    parishioner PII, still 100% Manus-dependent, with NO migration in flight and a
    DATA-RECOVERY RACE against Manus teardown (forms + uploaded files). This is the
    single most urgent item on the whole board.
  · legacy-hoopers — same Manus shape but LOW urgency (unshipped prototype; live LH
    site is Squarespace; its data is served by aster-sports). Frank's queued rebuild.

So the headline for the architect: the code is in good shape; the PLATFORM PLUMBING is
the emergency. St Patrick needs a data rescue THEN a new home, and astersports-web needs
its just-merged Railway migration verified live.
