0. WHAT THE DEEP PASS CHANGES vs THE BREADTH PASS
════════════════════════════════════════════════════════════════════════
The breadth pass said the apps were "healthy-to-routine." That holds at the architecture
level — but line-by-line surfaced a real bug surface the breadth grep structurally
COULD NOT see, plus several pre-onboarding auth/correctness holes in St Patrick. Two
findings were also LIVE-VERIFIED into fact (academy drift = 1 real player; payment-scope
drift = 2 real families). Net: the architecture is sound; the edges are not airtight, and
St Patrick specifically is NOT yet hardened.

Why breadth missed it [GROUNDED meta-finding]: the AP #36 audit test
(destructureWithoutErrorAudit.test.js) only flags the `{ data: x = [] }` default form —
NOT the bare `const { data } = await ...` form. A whole class of error-swallowing reads
uses the bare form, so the breadth grep reported "0 hits" while the deep read found a
cluster. RECOMMEND widening that audit test's grep (it's the cheapest systemic fix here).

1. CROSS-CUTTING THEMES (AP #58 synthesis) — ranked by leverage
════════════════════════════════════════════════════════════════════════

T1. TIMEZONE-LOCALITY — the dominant systemic weakness, and it's a TENANT-#2 TIME BOMB.
    Pattern: date/time computed in the runtime/browser-local zone instead of pinned to
    America/New_York. DORMANT today (Eastern-only pilot, NY users) — but it goes LIVE the
    moment St Patrick onboards (a UTC serverless host), or any non-Eastern/traveling
    admin uses the app. Instances found:
    aster-sports: digestPeriod.defaultPeriod (digestPeriod.js:16-26, local getDay/setHours
      → UTC cron drafts the wrong week); coach_roundup + family_guide event windows
      (coachRoundup.js:48, familyGuide.js:109 — UTC literal upper bound drops a Sat-night
      ET game from the briefing); ScheduleForLaterPicker.jsx:22 (scheduled-send stores
      browser-local wall-clock as the NY intent); ScheduleListSections.jsx:24 +
      DateGroupedList.jsx:10 (week-boundary partition); recurrenceHelpers.js:9,24
      (toISOString().slice off-by-one in positive-UTC-offset browsers).
    st-patrick: the `new Date(x.toLocaleString("en-US",{timeZone}))` idiom at 11 sites /
      7 files (zoneless re-parse → every getHours/getDay offset-wrong off-ET);
      liturgicalSeason.ts:97-110 (Good Friday + Holy Saturday render as Ordinary Time;
      late-Nov Advent mis-classified — the most liturgically conspicuous wrong days of
      the year); icsGenerator.ts:43 (Mass .ics stamps local wall-clock as UTC Z → "Add
      to Calendar" lands Mass ~4-5h off in subscribers' calendars); CcdManager.tsx:42
      (datetime-local round-trip corrupts EVERY CCD event time on edit); formDefaults.ts:41
      (CCD legal signatureDate is a UTC module-load snapshot); date-only→UTC day-shift
      across AllCalendars/SEO/calendarData.
    LEVERAGE: a single shared `nowInET()` / ET date-build helper per app kills most of
    this. For St Patrick the two liturgical one-liners + a boundary test are nearly free.

T2. ERROR-SWALLOWING ON DATA-INTEGRITY PATHS (AP #36, bare-`data` form).
    Reads that drop `error` and substitute `[]`/null, rendering "empty" indistinguishably
    from "failed." Worst instances touch money/comms:
    aster-sports: leagueAppsImport.js:79 (unchecked guardians read → dedup maps go empty →
      EVERY family re-inserts: duplicate guardians + RE-POSTED fee/payment transactions on
      a re-run — money path); tournamentRosters.js:8,21 (unchecked read silently drops a
      team from the CANONICAL tournament-audience seed → those kids miss every tournament
      briefing); wizard StepDetails/StepTeam/CreateActivityWizard; the tournament-tab read
      hooks (GamesTab/ScenariosTab/MessagesTab/RosterTab); useGameResultsMap.js:22 (throw
      in uncaught .then → unhandled rejection); useDmThreads.js:25 spine query.
    st-patrick: swallowed mutation errors with no onError (CcdPermissionsManager,
      MassIntentionsManager, VolunteerNeedsManager, VolunteerManager resets form on
      failure), routeNotification HTML-injection-adjacent unescaped fields.
    LEVERAGE: widen the AP #36 audit test (the meta-fix in §0), then sweep the flagged
    callsites. The two import/seed utilities are the priority (money + comms).

T3. SAME-CONCEPT DIVERGENCE (AP #63) — still the platform's signature bug class.
    aster-sports [LIVE-VERIFIED]: academy roster_type(per-team) vs member_type(global) —
      1 real player mis-gated today; payment "owes?" season-scoped dot vs all-seasons

      families read opposite today. Plus: useHasUnread.js:30 (badge reads phantom
      `global_last_check` key no writer sets → /messages badge NEVER clears);
