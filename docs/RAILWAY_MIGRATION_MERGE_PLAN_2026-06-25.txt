4. RECOMMENDED MERGE PLAN (re-apply the migration layer onto current main)
-----------------------------------------------------------------------------------------
Do NOT open "#103 → main" as a 39-commit merge. Instead:

  Step 1 — Integration branch off CURRENT main:
     git checkout main && git pull
     git checkout -b chore/railway-migration-integration

  Step 2 — Bring the migration layer over. Two equivalent ways:
     (a) Cherry-pick the migration-unique commits from #103 (schema/driver, CAS port,
         migration-baseline+triggers, CI→Postgres, query-isms, LLM→Anthropic,
         storage→Supabase, auth→Google, cron scheduler, env guards, dead-Manus deletion),
         OR
     (b) Checkout the §3.A migration-unique paths wholesale from #103
         (`git checkout caa090f2 -- <paths>`), then hand-apply the §3.C reconciliations.
     (b) is faster given §3.A lands clean and §3.B needs nothing; reserve cherry-pick for
     files where main and #103 both moved (§3.C).

  Step 3 — Resolve the §3.C files by hand (union semantics): keep main's IA/authz/weather
     AND #103's Postgres/Google/Supabase wiring. Regenerate pnpm-lock via `pnpm install`.

  Step 4 — Green the gates locally and in CI:
     pnpm run check  (tsc)         -> clean
     pnpm run test   (vitest)      -> full suite green
     money-path integration (real Postgres) CI job -> green
     pnpm run build  (client+server) -> clean, bundle within budget

  Step 5 — Architect §2.4 sign-off (see §5) on the money-path CAS diff and the auth
     identity model. These HOLD for review; they do not auto-merge.

  Step 6 — Operator pre-cutover (see §6): Google OAuth redirect URIs, Railway env vars,
     fresh Supabase seed, ledger-reconciliation check, full smoke.

  Step 7 — Merge the integration branch to main. Now main carries BOTH the new design AND
     the working Railway+Supabase backend. Point Railway at main; retire the #103 pin.

  Step 8 — Cutover + decommission Manus (downtime cutover is acceptable pre-onboarding),
     keep Manus briefly as rollback, then tear it down.

WHY this beats merging the branch: it isolates the ~18-file migration delta from the
30-odd duplicated studio commits, eliminating the spurious self-conflicts entirely, and
it rebases the risky money-path/auth changes onto the freshest authz code (#110) so the
two §2.4 reviews look at the final combined state, not a stale snapshot.
