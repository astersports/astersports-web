    [GROUNDED] §1 below)
  · no agent flips a *_LIVE / mask-provider / sub-processor secret (Frank flips)

STEP 2/3 — CANDIDATES (each: what · our code · win · cost · privacy · GATE ·
invariant collision · lean)
--------------------------------------------------------------------------------

C1 — PERSISTENT SEGMENTATION CACHE + FREEZE-ON-BILL (the determinism moat)
  WHAT [GROUNDED]: replace the in-memory LRU (W2) with a persistent, content-keyed
    mask store (Supabase Postgres row or Storage object), keyed
    provider+modelVersion+imageHash+bbox(+seed). Once a garment is billed, its mask
    is frozen and never recomputed.
  OUR CODE [FIT]: server/_core/masking/segCache.ts (swap the Map for a persistent
    backend behind the same cachedSegmentation() signature).
  WIN: (a) cost — skip the dominant SAM2 spend on every re-run/restart, not just the
    last 4; (b) it is the PREREQUISITE for any SAM3 money-path flip (#184 A4 /
    freeze-on-bill); (c) guarantees an already-billed customer's output can't change.
  COST: moderate (a table/bucket + key derivation + read-through). New dep: none.
  PRIVACY: masks are derived from the customer image → stored at rest in OUR
    Supabase (already the storage boundary). No NEW third party. Retention/delete
    policy must be defined.
  GATE: 🔴 HUMAN — it underpins the money-path determinism contract (freeze-on-bill).
  INVARIANT COLLISION: none — it STRENGTHENS determinism + byte-stability. (Must
    preserve byte-identical read-through.)
  LEAN: PROTOTYPE (highest leverage; unblocks SAM3 and saves cost standalone).

C2 — ACTIVATE THE REPLICATE WEBHOOK (latency — config, not code)
  WHAT [GROUNDED]: the webhook code path is already implemented + env-gated (W3);
    ACTIVATE it by setting REPLICATE_WEBHOOK_SECRET + a public base URL so jobs
    complete on Replicate's callback instead of the ≤poll-interval cron latency.
  OUR CODE [FIT]: already wired — studio.ts:59 replicateWebhookUrl() →
    startSam2Segmentation → startPrediction; server/routes/replicateWebhook.ts is
    the fail-closed HMAC handler. No new wiring; this is deploy config + verify.
  WIN: completion latency drops from "next poll tick" to "on inference done"; the
    cron stays as the backstop. Billing UNCHANGED.
  COST: low code; BLOCKED on a public base URL + REPLICATE_WEBHOOK_SECRET (the
    domain/DNS situation — Frank's, not CC's).
  PRIVACY: no new third party (same Replicate). Webhook is fail-closed HMAC.
  GATE: 🟡 BENCH/architecture — not money (billing unchanged); verify the cron
    backstop still refunds a stranded job (the Gate-B proof).
  INVARIANT COLLISION: none. LEAN: PILOT once a public URL exists (sequence after
    the domain is settled).

C3 — DEPENDENCY / SUPPLY-CHAIN HYGIENE (prune residue)
  WHAT [GROUNDED]: remove mysql2, @aws-sdk/client-s3 + @aws-sdk/s3-request-presigner,
    and vite-plugin-manus-runtime IF a grep proves zero imports (W4).
  OUR CODE [FIT]: package.json + an import grep across server/ client/.
  WIN: smaller install + attack surface (airtight-build §6 Dependabot-at-0), less
    confusion (no MySQL driver in a Postgres app).
  COST: trivial; risk = a hidden import — gate the PR on `grep` proving zero refs.
  PRIVACY: none. GATE: 🟢 ROUTINE. INVARIANT COLLISION: none.
  LEAN: PROTOTYPE (cheap; do the grep first, then a routine prune PR).

C4 — REACT COMPILER 1.0 (client auto-memoization)
  WHAT [GROUNDED/web 2026-06-28]: React Compiler 1.0 went stable late 2025 — a
    build-time Babel/Vite plugin that auto-memoizes React 19 components, removing
    most manual useMemo/useCallback. [infoq.com/news/2025/12/react-compiler-meta/]
  OUR CODE [FIT]: client/ (React 19.2 + Vite 7) — the Studio editor
    (StudioEditor.tsx, ControlPanel, polling hooks) is the interactivity-sensitive
    surface.
  WIN: smoother editor interaction + less manual memo maintenance; build-time only.
  COST: add the Vite plugin; build-time compile cost. New devDep.
  PRIVACY: none (build tool). GATE: 🟡 — it changes RENDER behavior; pilot with a
    visual/interaction regression check before merge (must not change UI output).
  INVARIANT COLLISION: none on the deterministic SERVER ops (client-only). Honest-
    claims unaffected. LEAN: PILOT (enable on a branch, eyeball the editor + key
    flows, keep if no regressions).

C5 — SERVER OBSERVABILITY ON THE MONEY/ASYNC PATH
  WHAT [GROUNDED]: no error-tracking dep exists (W5); §7 lists it unconfirmed. Add
    server-side error tracking (e.g. Sentry node) scoped to the async worker, cron,
    and money path.
  OUR CODE [FIT]: studioAsyncWorker.ts, scheduler.ts, studio.ts deduct/refund.
  WIN: prod visibility into stranded jobs / refund failures / cron 403s (the class
    that bit us at #88) — currently only server_logs.
  COST: low-moderate; new dep + DSN secret. PRIVACY: scrub PII — do NOT send
    customer image URLs/payloads to the tracker (server errors only); that scrubbing
    must be configured FIRST (mirrors the aster-sports Sentry beforeSend discipline).
  GATE: 🟡 — third-party egress of error data; privacy scrub before enabling.
  INVARIANT COLLISION: none if payloads are scrubbed. LEAN: PROTOTYPE (real §7 gap).

C6 — DEPLOY-TIME MIGRATIONS (close W6)
  WHAT [GROUNDED]: the Railway release step does not run drizzle-kit migrate (W6) —
    prod drifted behind main this session. Add a gated release migration step OR a
    CI "pending-migrations" guard.
  OUR CODE [FIT]: deploy/release config + drizzle-kit migrate.
  WIN: schema stays in lockstep with main; no silent "table missing in prod."
  COST: low. PRIVACY: none. GATE: 🔴 HUMAN — it auto-applies schema to prod
    (owner-applied-only invariant); design it as owner-gated, not a blind auto-run.
  INVARIANT COLLISION: must NOT violate "migrations owner-applied only" — so the
    recommendation is a CI DRIFT GUARD (fail if main has unapplied migrations) +
    a one-command owner apply, NOT a blind auto-migrate on deploy.
  LEAN: PROTOTYPE the drift-guard (safe); the auto-apply itself stays owner's call.
